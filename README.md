# 유니뷰 필터 (경희대 공지 필터링 도구)

경희대 공지 게시판(일반/학사/장학/근로/시간표변경/교내학점교류/행사)을 직접 크롤링해서,
카테고리·키워드·개별 글 단위로 필요 없는 소식을 숨길 수 있게 해주는 도구입니다.

## 구성

- `crawler.js` — 게시판을 긁어서 `data/items.json`에 저장하는 스크립트
- `index.html` — 결과를 보여주는 화면 (카테고리 끄기, 키워드 차단, 개별 글 ✕ 삭제)
- `data/items.json` — 지금은 미리보기용 샘플 데이터 3건이 들어있음. 크롤러를 돌리면 덮어씌워집니다.

## 1. 로컬에서 미리보기 (지금 바로 가능)

크롤러 없이도 화면이 어떻게 생겼는지 먼저 볼 수 있어요.

```bash
cd uniview-filter
npx serve .
```

브라우저에서 열리는 주소로 들어가면 샘플 데이터 3건이 보이고, 카테고리 버튼 끄기 / 키워드 추가 /
개별 글 ✕ 삭제가 실제로 작동합니다 (브라우저의 localStorage에 저장되어 새로고침해도 유지돼요).

## 2. 실제 크롤링 돌리기

```bash
cd uniview-filter
npm install
node crawler.js
```

성공하면 `data/items.json`이 실제 공지사항으로 교체됩니다. 그 상태에서 다시 `npx serve .`로 열면
진짜 데이터가 보여요.

**⚠️ 만약 크롤링 시 403 에러가 난다면:**
경희대 서버가 브라우저가 아닌 요청을 막고 있는 걸 수도 있어요. `crawler.js`의 `fetchListPage` 함수
안 `axios.get` 부분에 아래처럼 `Referer` 헤더를 추가해보세요.

```js
headers: {
  "User-Agent": "Mozilla/5.0",
  "Referer": "https://www.khu.ac.kr/kor/user/bbs/BMSR00040/list.do?menuNo=" + menuNo,
},
```

## 3. 다음 단계 (자동화)

지금은 `node crawler.js`를 수동으로 실행해야 데이터가 갱신돼요. 매번 실행하지 않고 자동으로
최신 상태를 유지하려면:

1. 이 폴더를 GitHub 저장소로 올리기
2. **GitHub Actions**로 `node crawler.js`를 하루 1~2번 자동 실행하고, 결과를 저장소에 커밋하도록 설정
3. **GitHub Pages**로 `index.html`을 무료로 배포 → 휴대폰 브라우저에서 즐겨찾기 해두면 사실상 나만의 유니뷰 앱처럼 사용 가능

이 부분(GitHub Actions 자동화 + Pages 배포)은 실제로 저장소를 만드신 뒤 같이 설정하는 게 정확해요.
준비되시면 말씀해주세요.

## 참고: 삭제 규칙이 저장되는 방식

지금 버전은 **브라우저의 localStorage**에 숨김 목록을 저장해요. 즉:
- 같은 브라우저에서는 계속 유지됨
- 다른 기기/브라우저로 옮기면 초기화됨 (아직 서버 저장소가 없어서)

여러 기기에서 동기화하고 싶으시면 나중에 간단한 저장소(예: 파일 기반 DB나 Google Sheet)를
추가하는 방향으로 확장할 수 있어요.
